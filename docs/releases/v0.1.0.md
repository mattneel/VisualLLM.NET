# VisualLLM.NET v0.1.0

VisualLLM.NET is an OpenAI-compatible inference server implemented in Visual Basic .NET, with an in-process native backend for GGUF-based local models.

This is the first release where the repository stops being a proxy-shaped bit and starts owning runtime behavior directly. The managed server now resolves and loads `llama.cpp` in-process via P/Invoke, stages portable native dependencies under `runtimes/<rid>/native`, and ships release archives intended for other people, not just the workstation that built them.

## Highlights

- In-process `llama.cpp` backend through native P/Invoke bindings.
- OpenAI-compatible `/v1/models` and `/v1/chat/completions` endpoints, including SSE streaming.
- Portable packaging scripts for `win-x64`, `linux-x64`, and `osx-arm64`.
- GitHub Actions CI and tag-driven release automation.
- Release artifacts accompanied by SHA256 checksum files.

## Release Contents

Each archive contains:

- The published VisualLLM.NET server executable and managed assemblies.
- The native `llama.cpp` runtime staged under `runtimes/<rid>/native`.
- `LICENSE` and `CONTRIBUTING.md`.

Each archive does not contain:

- Model weights.
- Build caches.
- Debug symbol files.
- Machine-local toolchain debris.

## Operational Notes

- Hardware acceleration support is inherited from how `llama.cpp` was built for the target runtime.
- The release smoke test uses a tiny GGUF suitable for validation, not for bragging rights.
- If you are looking for Python, there is none in the serving path. This is intentional.
