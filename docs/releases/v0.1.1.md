# VisualLLM.NET v0.1.1

VisualLLM.NET remains an OpenAI-compatible inference server implemented in Visual Basic .NET, with an in-process native backend for GGUF-based local models. The `v0.1.1` cut is smaller than `v0.1.0`, but more operationally serious in exactly the way a release engineer would regret encouraging.

## Highlights

- Publishes `VisualLLM.Inference` to GitHub Packages for consumers that want the OpenAI-style contract types and managed inference abstractions without cloning the repository.
- Keeps the binary release flow intact for `win-x64`, `linux-x64`, and `osx-arm64`.
- Verifies NuGet packaging in CI before the tag workflow tries to publish it.
- Links release and package metadata to the actual repository instead of an aspirational one.

## Package Notes

- Package ID: `VisualLLM.Inference`
- Registry: GitHub Packages NuGet feed for the repository owner namespace
- Intended contents: contract types, runtime settings, prompt helpers, and inference abstractions
- Not included: model weights, native `llama.cpp` binaries, or the ASP.NET Core server host

## Operational Notes

- Binary runtime validation remains strongest on `win-x64`; Linux and macOS continue to rely on automation-first confidence.
- GitHub Packages defaults new packages to private visibility on first publish. If public consumption is the goal, adjust package visibility accordingly after the initial push.
- The server still runs in-process through P/Invoke. The package work does not dilute that decision; it just makes the managed surface less monastic.
